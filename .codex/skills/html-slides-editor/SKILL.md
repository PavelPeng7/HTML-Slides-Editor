# HTML Slides Editor

Use this skill when the user wants AI-generated HTML slides to be manually editable **inside the HTML preview itself**. Do not build a Chrome extension and do not frame the work as an HTML checker. The deliverable is an HTML slides/page that includes an embedded editor runtime.

Before modifying the runtime, switch script, autosave behavior, or public packaging, read `references/spec.md`. Treat it as the source of truth and verify against its regression checklist before reporting the work done.

## What To Build

When this skill is invoked, add a built-in editing layer to the target HTML:

- A fixed top banner appears in the preview.
- The banner shows `Editing` when editing is active.
- The banner includes undo, redo, and stop-editing controls.
- While editing is active, the user can click rendered text and type directly, like editing a document.
- While editing is active, the user can drag an image file onto an existing image/visual in the HTML preview and replace it.
- Pausing editing restores normal page/slides interaction.

Use `scripts/slides_editor_switch.py` for normal on/off operations. With `--autosave`, it copies `assets/html-slides-editor-runtime.js`, `assets/html-slides-editor-autosave.html`, and `assets/html-slides-editor-server.js` beside the slides and adds or removes the editor tags idempotently.

## Integration Workflow

1. Find the target HTML file generated by the AI Agent.
2. Enable the editor with persistent saving unless the user explicitly wants a throwaway preview:
   - `python3 .codex/skills/html-slides-editor/scripts/slides_editor_switch.py enable --autosave path/to/index.html`
   - Use `disable` instead of `enable` to turn it off.
   - If no path is provided, the script targets the sample slides in this repository.
3. Start the generated save server from the HTML file's folder:
   - `node path/to/html-slides-editor-server.js 8765`
4. Open the server URL, not `file://` and not a static server, when edits need to persist.
5. Verify the banner is visible above the slides content.
6. Verify text editing, undo, redo, stop editing, image drag-replacement, and reload persistence.

## Fixed Switch Commands

For this repository's sample slides:

```bash
python3 .codex/skills/html-slides-editor/scripts/slides_editor_switch.py enable --autosave
python3 .codex/skills/html-slides-editor/scripts/slides_editor_switch.py disable
```

For another HTML slides file:

```bash
python3 .codex/skills/html-slides-editor/scripts/slides_editor_switch.py enable --autosave path/to/index.html
python3 .codex/skills/html-slides-editor/scripts/slides_editor_switch.py disable path/to/index.html
```

The switch is safe to run repeatedly: enabling twice will not insert duplicate script tags, and disabling twice is a no-op.

## Runtime Behavior

The embedded runtime must follow `references/spec.md`. Core behavior:

- Set `document.designMode = "on"` during editing so ordinary rendered text can be edited directly.
- Keep the editor banner itself `contentEditable="false"`.
- Record undo history before text changes and before image replacements.
- Coalesce bursts of typing into one undo step.
- Stop page/slides click handlers while editing so links, buttons, slide navigation, and scripted click actions do not steal the interaction.
- Preserve common slides keyboard navigation after editing is stopped.
- Highlight the replaceable image/visual under the drag pointer.
- Replace these visual targets:
  - `<img>`,
  - `<picture>` fallback `<img>`,
  - inline `<svg>`,
  - SVG `<image>`,
  - elements with CSS `background-image`.
- Keep replacement images inside the original displayed image area; cover-fill and clip overflow.
- Show the crop/resize handle at the bottom-right after image replacement.
- Protect images from text-caret placement and accidental Backspace/Delete removal while editing.

## Persistence

The browser cannot write back to disk from `file://` or a static server. Persistent editing requires the generated `html-slides-editor-server.js` so the page can POST to `/save`.

Use `enable --autosave` for durable edits. The autosave snippet waits 1.8 seconds after a text, image, or crop edit, then POSTs a cleaned clone of the document to `/save`. The server writes that clone back to `index.html`.

The clean clone keeps user edits such as text, image `src`, SVG image `href`, background image URLs, and crop data. It strips temporary editor chrome before writing: `#__html_presentation_editor_bar`, runtime style tags, drop overlays, crop handles, autosave labels, drag highlight attributes, crop-active classes, and transient body classes.

Image replacements must use persistent `data:` URLs, not `blob:` object URLs, because `blob:` URLs expire on reload.

### Known issue: designMode and slide navigation

When `document.designMode = "on"` is active, `<button>` elements inside the editable document lose their click-event behavior. To protect slide navigation:

- Add `contenteditable="false"` to the nav container: `<nav id="nav" ... contenteditable="false">`.
- Use `element.onclick = handler` (property assignment) instead of `addEventListener` for nav buttons — this is more robust under designMode than `addEventListener`.

```html
<nav id="nav" aria-label="Slides" contenteditable="false">
  ...
</nav>
<script>
  function wireNav() {
    dots.forEach((dot, i) => { dot.onclick = () => go(i); });
  }
  wireNav();
</script>
```

## Presentation Compatibility

For HTML slides with slide navigation, animation, or WebGL/canvas backgrounds:

- Do not edit canvas pixels; edit DOM text and replaceable DOM visuals only.
- Suppress navigation interactions only while editing is active.
- Avoid changing the slides runtime unless needed for editor compatibility.
- Keep the banner above everything with a high `z-index`.
- Add top padding/margin only if the banner covers important content; prefer a compact banner.

## Quality Bar

Before finishing, verify the regression checklist in `references/spec.md`, including:

- Banner appears at the top of the HTML preview.
- `Editing` state is visible when active.
- Stop editing turns direct text editing off.
- Undo/redo work for typed text.
- Dragging an image over an existing visual highlights the target.
- Dropping the image replaces the intended visual.
- Dropped images keep the old visual frame, cover-fill it, and show the crop handle.
- Images cannot be removed accidentally with Backspace/Delete in editing mode.
- Normal slides interaction (pagination, keyboard nav) returns after editing stops.
- If auto-save is enabled: editing triggers a save after ~1.8 s, "Saved" appears in the bar and fades out, and the change survives a page reload.

## Output Style

When reporting back:

- Explain that the editor is embedded in the HTML preview, not a Chrome extension.
- Mention how the user edits text and replaces images.
- Be explicit about whether changes are preview-only or exportable/persistent.
- Mention the files changed.
