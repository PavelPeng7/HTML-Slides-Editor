# HTML Slides Editor

Use this skill when the user wants AI-generated HTML slides to be manually editable **inside the HTML preview itself**. Do not build a Chrome extension and do not frame the work as an HTML checker. The deliverable is an HTML slides/page that includes an embedded editor runtime.

## What To Build

When this skill is invoked, add a built-in editing layer to the target HTML:

- A fixed top banner appears in the preview.
- The banner shows `Editing` when editing is active.
- The banner includes undo, redo, and stop-editing controls.
- While editing is active, the user can click rendered text and type directly, like editing a document.
- While editing is active, the user can drag an image file onto an existing image/visual in the HTML preview and replace it.
- Pausing editing restores normal page/slides interaction.

Use `scripts/slides_editor_switch.py` for normal on/off operations. It copies `assets/html-slides-editor-runtime.js` beside the slides and adds or removes the script tag idempotently.

## Integration Workflow

1. Find the target HTML file generated by the AI Agent.
2. Enable the editor with the switch script:
   - `python3 .codex/skills/html-deck-editor/scripts/slides_editor_switch.py enable path/to/index.html`
   - Use `disable` instead of `enable` to turn it off.
   - If no path is provided, the script targets the sample slides in this repository.
3. Ensure the runtime starts in editing mode by default unless the user asks for a separate start button.
4. Verify the banner is visible above the slides content.
5. Verify text editing, undo, redo, stop editing, and image drag-replacement.

## Fixed Switch Commands

For this repository's sample slides:

```bash
python3 .codex/skills/html-deck-editor/scripts/slides_editor_switch.py enable
python3 .codex/skills/html-deck-editor/scripts/slides_editor_switch.py disable
```

For another HTML slides file:

```bash
python3 .codex/skills/html-deck-editor/scripts/slides_editor_switch.py enable path/to/index.html
python3 .codex/skills/html-deck-editor/scripts/slides_editor_switch.py disable path/to/index.html
```

The switch is safe to run repeatedly: enabling twice will not insert duplicate script tags, and disabling twice is a no-op.

## Runtime Behavior

The embedded runtime should:

- Set `document.designMode = "on"` during editing so ordinary rendered text can be edited directly.
- Keep the editor banner itself `contentEditable="false"`.
- Record undo history before text changes and before image replacements.
- Coalesce bursts of typing into one undo step.
- Stop page/slides click handlers while editing so links, buttons, slide navigation, and scripted click actions do not steal the interaction.
- Preserve common slides keyboard navigation after editing is stopped.
- Highlight the replaceable image/visual under the drag pointer.
- Replace these visual targets:
  - `<img>`,
  - `<picture>` fallback `<img>`,
  - inline `<svg>`,
  - SVG `<image>`,
  - elements with CSS `background-image`.

## Persistence

By default, this editor changes the live preview DOM. That is enough for quick manual polish during a preview session, but it does not automatically update the source file after reload.

If the user needs durable output, add an explicit save/export feature:

- Export current `document.documentElement.outerHTML` after removing transient editor highlights.
- Keep the embedded editor runtime in the exported file unless the user asks for a clean final HTML.
- For image replacements from local files, use Data URLs instead of object URLs when durable export is required.

## Presentation Compatibility

For HTML slides with slide navigation, animation, or WebGL/canvas backgrounds:

- Do not edit canvas pixels; edit DOM text and replaceable DOM visuals only.
- Suppress navigation interactions only while editing is active.
- Avoid changing the slides runtime unless needed for editor compatibility.
- Keep the banner above everything with a high `z-index`.
- Add top padding/margin only if the banner covers important content; prefer a compact banner.

## Quality Bar

Before finishing, verify:

- Banner appears at the top of the HTML preview.
- `Editing` state is visible when active.
- Stop editing turns direct text editing off.
- Undo/redo work for typed text.
- Dragging an image over an existing visual highlights the target.
- Dropping the image replaces the intended visual.
- Normal slides interaction returns after editing stops.

## Output Style

When reporting back:

- Explain that the editor is embedded in the HTML preview, not a Chrome extension.
- Mention how the user edits text and replaces images.
- Be explicit about whether changes are preview-only or exportable/persistent.
- Mention the files changed.
